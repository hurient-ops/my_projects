| **데이터 연동** | BLE 통신을 통한 인슐린 펌프 실시간 데이터 수신 (최소 요구사항) | 높음 (핵심 가치 제공) |

| **사용자 인터페이스** | 실시간 상태 표시 및 알림의 직관적인 UI/UX 제공 | 중간 (사용자 채택률 결정) |
| **데이터 로깅** | 모든 경고 이벤트 및 시스템 상호작용에 대한 타임스탬프 기반 로그 기록 (규제 준수 목적) | 높음 (법적 방어 및 검증 자료) |

## 2. 초기 사용자 확보 전략 (Go-to-Market Strategy)

초기 시장 진입은 '규제 준수'와 '신뢰 구축'에 초점을 맞춥니다.

1.  **B2B/병원 연계 선행 접근 (Pilot Program):**
    *   **전략:** 소규모 당뇨 클리닉 또는 병원과의 파일럿 프로그램을 통해 실제 임상 환경에서의 데이터 안정성과 규제 준수(SaMD 프레임워크)를 검증받습니다. 이는 초기 신뢰도를 극대화합니다.
    *   **KPI:** 파일럿 참여 기관 수, 임상 피드백의 질.

2.  **개인 사용자 커뮤니티 기반 마케팅 (Trust Building):**
    *   **전략:** 당뇨환자 커뮤니티(온라인 포럼, 전문 그룹) 내에서 '안전성'과 '데이터 보안'을 핵심 메시지로 홍보합니다. 단순한 기능 제공이 아닌, **"예측 가능성과 안전망"**이라는 가치에 집중하여 접근합니다.
    *   **KPI:** 초기 무료 사용자 확보 수, 이탈률(Churn Rate).

## 3. 최적의 가격 모델 분석 (Pricing Model Analysis)

경쟁사 ROI 분석 결과와 타깃 고객(환자 및 의료기관)의 민감도를 고려하여 두 가지 모델을 제시합니다.

### **모델 A: 구독 기반 프리미엄 모델 (Subscription - B2C/개인 사용자 중심)**

*   **구조:** 월별 또는 연간 구독료 부과.
*   **가격 책정:**
    *   **Basic Tier ($19.99/월):** 기본적인 실시간 알림 및 데이터 로깅 제공. (초기 시장 침투용)
    *   **Premium Tier ($49.99/월):** 고급 예측 알고리즘(미래 위험 구간 예측), 상세 리포트 생성, 무제한 데이터 저장 공간 제공. (가치 기반 가격 책정)
*   **ROI 분석:** 초기 사용자 확보 비용(CAC)이 높을 수 있으나, 높은 LTV(고객 생애 가치)를 통해 장기적인 수익 안정성을 확보할 수 있습니다. **(안정적 성장 지향)**

### **모델 B: 서비스 기반 라이선스 모델 (Licensing - B2B/의료기관 중심)**

*   **구조:** 의료기관 또는 병원에 소프트웨어 사용 권한을 연간 또는 사용자당 라이선스 비용 부과.
*   **가격 책정:**
    *   **License Fee ($5,000 - $15,000/년):** 특정 환자군 관리 솔루션으로서의 통합 및 규제 준수 보고서 제공을 포함한 포괄적 라이선스.
*   **ROI 분석:** 초기 매출 규모는 작으나, 계약당 가치가 매우 높고(High-Value Deal), 의료기관이라는 확실한 고객 기반으로 인해 대형 계약을 통해 높은 마진을 확보할 수 있습니다. **(고마진/대규모 수익 지향)**

## 2. 기술 스펙: BLE 통신 프로토콜 상세 분석

### 2.1. 필수 통신 프로토콜 및 스펙 (개발 관점)

| 항목 | 요구사항/표준 | 세부 고려 사항 |
| :--- | :--- | :--- |
| **프로토콜** | BLE (Bluetooth Low Energy) | 에너지 효율성, 낮은 지연 시간(Latency) 확보. |
| **데이터 형식** | 표준화된 페이로드 정의 | 인슐린 수치, 센서 데이터, 알림 상태 등 필수 데이터를 JSON 또는 CBOR 형태로 구조화. |
| **보안 계층** | BLE Security (LE Secure Connections) | **AES-CCM** 암호화 사용 의무화. 연결 설정 시 강력한 키 교환 메커니즘 적용. |
| **데이터 무결성** | MAC/CRC 검증 | 전송되는 모든 데이터 패킷에 대한 오류 검증 로직 구현 (신뢰성 확보). |
| **통신 안정성** | 재연결 및 상태 관리 | 연결 끊김 발생 시, 세션 상태를 안전하게 저장하고 재연결 시 이전 상태 복구 메커니즘 설계. |

### 2.2. 보안 요구사항 (규제 준수 관점)

의료기기 소프트웨어(SaMD)로서 데이터의 기밀성 및 무결성은 법적 요구사항보다 우선합니다.

*   **데이터 암호화:** BLE 통신 채널 전체에 대한 종단 간 암호화(End-to-End Encryption)를 구현해야 합니다. 개인 식별 정보(PHI)가 포함될 경우, 전송 중뿐만 아니라 저장된 데이터 역시 강력하게 암호화되어야 합니다.
*   **접근 제어:** 펌프 장치와 앱 간의 통신에 대해 최소 권한 원칙(Principle of Least Privilege)을 적용하여, 비인가 접근 시도를 차단해야 합니다.
*   **보안 감사 로그:** 모든 데이터 송수신 및 시스템 상태 변경 기록을 암호화된 형태로 로깅하여 추후 규제 기관 감사에 대비해야 합니다.
