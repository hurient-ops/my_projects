MVP의 핵심은 **'위험 구간 알림 기능'**과 **'BLE 통신 보안/무결성 확보'**입니다. 이 두 가지 목표를 달성하기 위한 기술 스택 정의부터 구현까지의 로드맵을 설계합니다.

### 🚀 4주 개발 마일스톤 프레임워크 초안

이 프레임워크는 **기술 안전성(Security & Compliance)**을 모든 단계에 내재화하는 것을 최우선 원칙으로 설정했습니다.

#### 🎯 핵심 목표 (MVP 정의)
**"인슐린 펌프의 실시간 데이터를 BLE를 통해 안전하게 수신하고, 임상적으로 타당한 안전 알림 기능을 제공한다."**

---

### 🗓️ 주차별 계획 상세 (4 Weeks Sprint)

| 주차 | 목표 (Focus Area) | 주요 기능 (Scope) | 핵심 기술 요구사항 (Technical Requirement) | 산출물 목표 |
| :--- | :--- | :--- | :--- | :--- |
| **Week 1** | **기반 설계 및 보안 아키텍처 확정** | 프로젝트 구조 설정, 데이터 모델 정의(DB/API), BLE 통신 프로토콜 스펙 최종 확정. | SaMD 프레임워크 초기 설계, BLE Security (AES-CCM) 구현 방안 상세 정의, 인증/인가 메커니즘 설계. | 시스템 아키텍처 다이어그램 및 보안 설계 문서 초안 |
| **Week 2** | **BLE 통신 모듈 개발 (연결 및 데이터)** | 펌프 장치와의 초기 BLE 연결 및 실시간 데이터 스트리밍 구현. | 안정적인 LE Secure Connections 설정, 데이터 패킷의 MAC/CRC 무결성 검증 로직 통합, 상태 관리(Connection State) 구현. | BLE 통신 프로토콜 핸들러 (PoC 수준) |
| **Week 3** | **핵심 안전 로직 개발 (알림)** | 수신된 데이터 기반으로 위험 구간 판단 및 알림 생성 알고리즘 구현. | 임상적으로 타당한 임계값 설정 모듈, 오탐 방지 안전장치(Safety Margin) 로직 통합, 알림 우선순위 시스템 설계. | 안전 알림 로직 백엔드 모듈 완성 |
| **Week 4** | **프론트엔드 연동 및 검증 (MVP)** | 사용자 인터페이스(UI) 구현 및 BLE 데이터 시각화, 전체 흐름 테스트 및 규제 준수 점검. | `mobile-kit` 기반 UI/UX 통합, 데이터 암호화된 저장소 접근 로직 검증, End-to-End 보안 경로 최종 검증. | MVP 기능 통합 데모 빌드 (Internal Alpha Test) |

### ⚙️ 기술 및 비즈니스 연계 검토 사항

1.  **기술 안전성(Researcher 반영):** Week 1에서 정의된 **보안 아키텍처**는 모든 후속 개발의 기반이 됩니다. 특히 BLE 통신 채널 전체에 대한 종단 간 암호화와 데이터 무결성 검증은 단순한 기능 추가가 아닌, **MVP 성공의 전제 조건**으로 취급되어야 합니다.
2.  **비즈니스 가치(Business 반영):** Week 3에서 구현할 **위험 알림 로직**은 '예측 가능성 및 안전망'이라는 프리미엄 가치를 실현하는 핵심입니다. 알고리즘 설계 시, 임상적 타당성 검토가 개발 단계부터 병행되어야 합니다 (Researcher의 리서치 결과 활용).
3.  **리소스 분배 제안:** 이 프레임워크를 실행하기 위해 **Developer 에이전트**에게는 Week 1과 Week 2에 가장 많은 자원을 할당하여 아키텍처와 핵심 통신 모듈을 완성하도록 지시하겠습니다.
