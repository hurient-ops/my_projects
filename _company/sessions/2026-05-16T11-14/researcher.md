### 📝 Researcher 산출물: 데이터 무결성 및 감사 로그 기술적 증빙 확보 방안 (초안)

개발팀이 구현할 코드가 규제 준수를 입증하는 '증거'가 되도록, 다음 세 가지 핵심 영역을 중심으로 기술적 증빙 자료 확보 전략을 제시합니다.

#### 1. 데이터 무결성(Hashing/Signing)에 대한 기술 증빙 전략

**목표:** 전송 및 저장된 모든 민감 데이터의 무결성이 보장됨을 입증해야 합니다.

| SaMD 체크리스트 항목 (예시) | 요구되는 기능 (What) | 필요한 코드 구현 요소 (How - 코다리에게 요청할 내용) | 기술적 증빙 자료 확보 방안 (Evidence) |
| :--- | :--- | :--- | :--- |
| **데이터 무결성 보장** | 송수신 데이터의 변조 여부 확인 | BLE 통신 시 Payload에 대한 HMAC(Hash-based Message Authentication Code) 생성 및 검증 로직 구현. | `auth_crypto.ts` 내에서 생성된 Hash/Signature 값과, 실제 전송된 데이터 및 수신된 데이터 간의 일치성을 검증하는 테스트 코드(`test_hashing.ts`) 또는 모듈 사용 예시. |
| **데이터 기밀성 보장** | 민감 정보(혈당 데이터 등)의 암호화 | 데이터 저장 시 AES-256-GCM을 사용한 엔드-투-엔드 암호화 구현. | 키 관리 방식(`SECRET_KEY`의 안전성 확보 방안), 암호화/복호화 함수 전체 코드, 그리고 암호화된 데이터가 DB에 저장될 때의 스키마 정의. |

#### 2. 감사 로그(Audit Log) 구축에 대한 기술 증빙 전략

**목표:** 시스템에서 발생한 모든 중요 이벤트(접근, 변경, 알림 발생 등)는 불변하게 기록되어야 합니다.

| SaMD 체크리스트 항목 (예시) | 요구되는 기능 (What) | 필요한 코드 구현 요소 (How - 코다리에게 요청할 내용) | 기술적 증빙 자료 확보 방안 (Evidence) |
| :--- | :--- | :--- | :--- |
| **변경 기록 추적** | 시스템 상태 변경 이력 기록 | 사용자 설정 변경, 위험 알림 발송 등 중요한 이벤트 발생 시, 타임스탬프와 사용자 ID를 포함하여 로그 레코드를 생성하는 함수 구현. | Audit Log 데이터베이스 스키마 정의(Table Schema), 로그 기록 시점의 불변성(Immutability)을 보장하기 위한 해시 체인 또는 블록체인 기반의 로그 무결성 검증 로직. |
| **접근 통제 기록** | 접근 시도 및 성공/실패 기록 | BLE 모듈에 대한 접근 시도 및 인증 실패 기록을 별도의 보안 로그로 분리하여 기록. | 접근 로그(`audit_log.json` 또는 DB 테이블)의 샘플 데이터와, 해당 로그가 시스템 외부에서 조작되지 않음을 증명하는 검증 메커니즘 설명. |
