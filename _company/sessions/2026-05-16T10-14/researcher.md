## 의료기기 규제 준수 상세 체크리스트: BLE 통신 및 위험 구간 알림 기능 검토

본 체크리스트는 개발팀이 정의한 **BLE 통신 보안 아키텍처**와 **위험 구간 알림 기능**이 관련 의료기기 규제(SaMD 프레임워크)를 준수하는지 확인하기 위해 설계되었습니다. 모든 항목은 현재의 기술 설계(P1, P2, P3 모듈)에 대해 '예/아니오'로 답변할 수 있도록 구성되어 있습니다.

### 섹션 1: 데이터 보안 및 통신 무결성 (Data Security & Integrity - SaMD 핵심 요구사항)

| No. | 검토 항목 (Requirement Focus) | 현재 설계 반영 여부 (Yes/No) | 상세 구현 기준 및 증빙 자료 요청 | 관련 규제/표준 (예시) |
| :---: | :--- | :---: | :--- | :--- |
| 1.1 | **데이터 암호화 (Encryption)** | | BLE 통신 채널(Link Layer) 및 상위 애플리케이션 데이터(Application Layer) 전송 시 사용된 알고리즘(예: AES-256)과 키 관리 방식은 규제에서 요구하는 수준인가? | HIPAA, GDPR, MDR Annex I |
| 1.2 | **인증 (Authentication)** | | 펌프 장치와 앱 간의 연결 시 상호 인증 메커니즘(예: DTLS/BLE Security Profile)이 구현되었으며, 무단 접근을 차단하는가? | IEC 80601-2-12 (전기적 안전성 관련 확장) |
| 1.3 | **데이터 무결성 (Integrity)** | | 전송되는 모든 인슐린 데이터 및 명령이 위변조되지 않았음을 보장하기 위해 해시 함수 또는 메시지 인증 코드(MAC)가 적용되었는가? | SaMD 요구사항 (Data Integrity) |
| 1.4 | **데이터 저장 보안** | | 펌프 장치와 앱에 저장되는 모든 민감 데이터(사용자 ID, 수치 기록 등)는 저장 시 암호화되어 있는가? | GDPR / HIPAA 준수 여부 확인 |
| 1.5 | **접근 통제 (Access Control)** | | 사용자 인증 및 권한 부여 시스템이 정의되었으며, 역할 기반 접근 제어(RBAC)를 통해 민감 데이터 접근을 제한하는가? | SaMD 보안 요구사항 |

### 섹션 2: 위험 구간 알림 기능의 임상적 타당성 및 안전성 (Risk Alert Validation - MVP 핵심 목표)

| No. | 검토 항목 (Requirement Focus) | 현재 설계 반영 여부 (Yes/No) | 상세 구현 기준 및 증빙 자료 요청 | 관련 규제/표준 (예시) |
| :---: | :--- | :---: | :--- | :--- |
| 2.1 | **임계값의 임상적 타당성** | | 위험 구간(알림 트리거)을 결정하는 임계값(Thresholds)이 실제 인슐린 주입 및 생리학적 데이터에 기반하여 의료 전문가의 검증을 받았는가? (오탐 방지 설계 포함) | MDR/FDA 요구사항 (Clinical Validity) |
| 2.2 | **오탐 방지 메커니즘** | | 시스템이 오경보(False Alarm)를 발생시키지 않도록 하는 안전장치(Safety Margin, Redundancy Check)가 명확하게 설계 및 구현되었는가? | IEC 62304 (소프트웨어 생명주기) |
| 2.3 | **알림의 중복성 및 우선순위** | | 위험 알림 발생 시, 알림의 중요도에 따라 사용자에게 전달되는 메시지의 우선순위와 중복 방지 로직이 정의되어 있는가? | 사용자 안전 설계 원칙 |
| 2.4 | **비상 상황 처리 (Fail-Safe)** | | BLE 통신 단절 또는 시스템 오류 발생 시, 위험 상태를 즉시 감지하고 안전 모드(Safe Mode)로 전환하거나 수동 개입을 요청하는 비상 프로토콜이 구현되었는가? | IEC 62304 (안전성 요구사항) |

### 섹션 3: 규제 준수 문서화 및 추적성 (Documentation & Traceability - SaMD 프레임워크)

| No. | 검토 항목 (Requirement Focus) | 현재 설계 반영 여부 (Yes/No) | 상세 구현 기준 및 증빙 자료 요청 | 관련 규제/표준 (예시) |
| :---: | :--- | :---: | :--- | :--- |
| 3.1 | **요구사항 추적성** | | 각 기능(보안, 알림)이 어떤 의료기기 규제 요구사항(섹션 1의 항목들)을 충족시키는지에 대한 명확한 매핑 테이블(Traceability Matrix)이 존재하는가? | ISO 14971 (위험 관리) |
| 3.2 | **설계 문서** | | BLE 통신 모듈 및 위험 알림 로직의 상세 설계 문서(Design Specification)가 작성되었으며, 이 문서에 모든 기술적 결정 사항이 명시되어 있는가? | SaMD 품질 시스템 요구사항 |
| 3.3 | **위험 분석 보고서 (Risk Analysis)** | | 시스템이 발생시킬 수 있는 잠재적 위험(예: 오탐으로 인한 과도한 조치, 데이터 유출)을 식별하고, 이를 완화하기 위한 조치가 문서화된 위험 분석 보고서가 존재하는가? | ISO 14971 |
